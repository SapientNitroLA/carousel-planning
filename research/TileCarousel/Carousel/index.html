tileCarousel Plugin for jQuery

Author: Ryan Fitzer and Travis Higdon

View a demonstration

Create an accessible JavaScript carousel out of any unordered list of tiles. Each tile should be placed in a list item `li` within a container `ul`. Additional markup is generated by JavaScript, and several required style definitions are included in a global stylesheet. These styles are external instead of plugin-generated so they can be overridden easily if neccessary.

Tiles can contain any html content but must be equal in width. Measurement for the overall carousel and individual tiles is based on the width of the first tile. Height can vary, although because the images are lazy loaded, image heights are evaluated just-in-time. This can lead to a slight page jump if a hidden tile is taller than the visible tiles once its images are loaded. CSS can be used to style the "previous" and "next" buttons, as well as the pagination menu (if pagination is enabled).

Instantiate the carousel(s) by calling the plugin on an element or elements and passing an optional options object.

Requires jQuery 1.4.2.

Plugin Options

  • increment (Number) Number of tiles to display per frame. Default is 1.
  • incrementMode (String) Whether to move the carousel by frame or single tile. Accepted values are `frame` and `tile`. Default is `frame`.
  • easing (String) Default is `swing`. Additional jQuery UI plug-in required for other easing options.
  • speed (Number) Speed of animation. Default is `750` miliseconds.
  • loop (Boolean) Default is `false`. If `true`, carousel will loop infinitely in either direction.
  • pagination (Boolean) Default is `false`. If `true`, pagination links are generated and can be styled.
  • centerPagination (Boolean) Calulates width of pagination wrapper and centers it beneath the carousel. Default is `true`.
  • encapsulateControls (Boolean) Default is `false`. If `true`, a wrapper is placed around the prev/next links and pagination and centered.
  • displayCount (Boolean) Default is `false`. If `true`, the current tile/frame context will be indicated (ie. "1-3 of 10").
  • accessible (Boolean) Default is `true`. Enables screenreader markup and keystroke control of carousel.
  • wrapperDelta (Number) Change wrapper width by this pixel value. Default is 0.
  • viewportDelta (Number) Change viewport width by this pixel value. Default is 0.
  • preFrameChange (Function) Callback fired before the transitional frame animation.
  • postFrameChange (Function) Callback fired after the transitional frame animation.

Code Examples

Example #1: Default Instantiation

Instantiation using default settings: single item carousel, one tile displayed at a time, advances one tile at a time, does not loop and does not display pagination. Note: the `carousel` class is not required for instantiation - any selector can be used.

$('.carousel').tileCarousel();

Example #2: Options

This example uses an `options` object to create a three-tile carousel that displays pagination and does not loop.

$('.carousel').tileCarousel({
     increment: 3,
     incrementMode: 'frame', 
     loop: false,
     pagination: true
});

Example #3: More Options

This example uses an `options` object to create a four-tile carousel that loops, does not display pagination and moves the carousel only one tile at a time (instead of the entire visible frame).

$('.carousel').tileCarousel({
     increment: 4,
     incrementMode: 'tile', 
     loop: true,
     pagination: false
});

Example #4: Using Callbacks

This example uses an `options` object to create a three-tile carousel that loops and triggers callback functions before and after the carousel frame changes. The two callbacks must be named `preFrameChange` and `postFrameChange`.

$('.carousel').tileCarousel({
     increment: 3,
     loop: true,
     callbacks: {
         preFrameChange: function() {
             // do something
         },
         postFrameChange: function() {
             // do something
         },
     }
});
